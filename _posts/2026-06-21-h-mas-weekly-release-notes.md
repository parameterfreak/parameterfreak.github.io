# H-MAS Iteration 27 작업 노트

이번 Iteration 27 기간(6월 14일 ~ 6월 20일) 작업 내용을 안내합니다. 이번 작업에는 21개 커밋, 18개 PR이 포함되어 있습니다.

하이라이트는 다음과 같습니다:

- HuggingFace 토큰을 전역 싱글턴에서 조직(Org) 단위로 분리하는 4-Phase 전환 완료.
- 배포 생성 시 HuggingFace Gated 모델을 사전 판정하여 토큰 미등록 조직의 배포를 사전 차단.
- 그룹/조직 삭제 시 산하 서빙 배포가 정리되지 않고 GPU 자원을 점유한 채 남던 좀비 워크로드 문제 해결.
- 조직/그룹 삭제 시 멤버십 레코드가 정리되지 않던 고아 레코드 문제 수정.
- 조직 삭제와 멤버 추가가 동시에 발생할 때 발생할 수 있던 고아 멤버십 race condition 방지.
- 그룹/조직 삭제 등 외부 사이드이펙트를 동반하는 작업을 위한 비동기 작업 큐 인프라 신설.
- 조직 관리자가 전역 런타임 이미지 설정 위에 자신의 조직만 이미지 버전을 재정의할 수 있는 오버라이드 기능 추가.
- 조직 생성 시 이메일 도메인 입력을 필수화하여 멤버십 도메인 정책 우회 차단.
- 프로필/사용자 관리 화면 간 역할 레이블 불일치 및 viewer 권한 손실 위험 수정.
- QoS만 지정되고 스케줄러가 미지정된 배포 요청에서 우선순위·QoS 설정이 적용되지 않던 버그 수정.
- 머지된 PR 패턴을 기준으로 PR 제목/본문 형식을 자동 검사하는 CI 워크플로 도입.
- 그 외 기술 문서 전반 최신화, 제품 소개 문서 갱신 등 자잘한 개선.
