# H-MAS Iteration 28 작업 노트

이번 Iteration 28 기간(6월 21일 ~ 6월 27일) 작업 내용을 안내합니다. 이번 작업에는 15개 커밋, 15개 PR이 포함되어 있습니다.

하이라이트는 다음과 같습니다:

- 멀티클러스터 환경에서 상위 리소스는 삭제됐으나 멤버 클러스터에 잔존하던 좀비 워크로드(고아 Work)를 주기적으로 감지·정리하는 자동 정리 기능을 도입하여, 누적되는 좀비 리소스와 control plane 부담 누수 경로를 차단(기본 비활성, 옵트인 출시).
- 시스템 관리자가 자동 정리 실패를 인지·조치할 수 있는 관리 UI 추가 — 정상 상태에서는 아무것도 노출하지 않는 무노이즈(zero-noise) 원칙 적용.
- 고아 워크로드 감지의 탐지 커버리지, 대규모 환경 확장성, 운영 로그 노이즈를 정리하는 후속 보강 3건 완료.
- 클러스터 등록·해제 권한을 고객사 조직 구성(중앙집중형/위임형)에 맞춰 선택할 수 있는 설정 가능한 온보딩 정책을 백엔드·프론트엔드 전 구간에 도입.
- 조직/그룹 삭제 확인 모달에 실제 연쇄 정리 규모(그룹·멤버·배포 수)를 사전 표시하여 대규모 오삭제 실수를 예방.
- 클러스터 페이지의 중복·오작동하던 조직 필터를 정리하여 조직 전환 경로를 단일화.
- 그 외 비동기 작업 큐·멀티테넌시 기술 문서 최신화, GPU 활용률 주장에 대한 근거 출처 정리, 사업계획서 작성 등 자잘한 개선.
